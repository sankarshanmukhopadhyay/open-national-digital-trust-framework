# Profile versioning and change control

Profiles are controlled governance instruments. Their versioning must communicate operational impact, not merely editorial history.

## Change classes

| Class | Example | Minimum treatment |
|---|---|---|
| editorial | correction without semantic effect | recorded change |
| clarifying | removes ambiguity without changing obligation | impact review |
| compatible | adds optional capability or stronger path | migration note |
| breaking | changes obligations, evidence, status, or interoperability | approval, transition, reassessment |
| emergency | immediate response to material harm or compromise | emergency authority, expiry, retrospective review |

Every breaking or emergency change must identify affected providers, relying parties, affected parties, dependent profiles, transition dates, continuity controls, and evidence required for closure.

A profile must publish effective date, superseded versions, support window, migration deadline, and retirement state. Emergency changes must not become permanent without ordinary governance review.
