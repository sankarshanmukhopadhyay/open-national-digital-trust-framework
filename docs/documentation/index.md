# Documentation Architecture

This section explains how the ONDTF body of knowledge is organised and how readers can move between its **learning**, **reference**, and **evidence** views.

| View | Purpose | Entry point |
|---|---|---|
| Learning | Acquire a coherent mental model in a deliberate sequence | [Learn ONDTF](../learning/index.md) |
| Reference | Find an authoritative definition, requirement, model, or workflow | [Documentation map](../index.md#documentation-map) |
| Evidence | Inspect worked artefacts and end-to-end traceability | [Worked reference scenario](../reference-scenario/index.md) |

## Documentation contract

Every substantial addition should identify where it sits in the framework, what prior concepts it depends upon, what it enables, and which learning paths require updating. New pages must not rely on sidebar position alone to communicate reading order.
