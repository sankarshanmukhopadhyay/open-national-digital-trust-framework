# Decision states and review gates

Every answer is recorded with provenance and a decision state.

| State | Meaning |
|---|---|
| selected | adopter made and owns the decision |
| inherited | accepted from an ONDTF or parent-profile default |
| profile-controlled | determined by an authorised subordinate profile |
| provisional | usable for drafting but not final approval |
| unresolved | no defensible decision has been made |
| review-required | competent legal, technical, assurance, accessibility, security, or stakeholder review is required |
| not-applicable | excluded with rationale, authority, and review condition |

Review gates are not comments. A blocking gate prevents the affected profile section or package from being marked approved.

Typical gates include legal authority, governance approval, security review, privacy review, accessibility review, assurance review, technical interoperability review, affected-party consultation, and operational evidence.
