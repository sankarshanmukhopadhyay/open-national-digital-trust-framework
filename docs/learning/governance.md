# Governance Path

**Audience:** Scheme authorities, governance designers, regulators, and oversight bodies  
**Prerequisites:** [ONDTF in One Hour](one-hour.md), or equivalent familiarity  
**Estimated time:** 2–3 hours  
**Outcome:** a role-specific understanding of the framework, with enough context to use the reference documentation confidently.

## Recommended sequence

| Order | Read | Why it comes here |
|---:|---|---|
| 1 | [Core principles](../foundations/principles.md) | Establish the governance posture. |
| 2 | [Governance framework](../governance/index.md) | Understand the federated model. |
| 3 | [Institutional functions](../operating-model/institutional-functions.md) | Identify required functions. |
| 4 | [Decision rights](../operating-model/decision-rights.md) | Allocate authority and accountability. |
| 5 | [Authority and delegation](../governance/authority-delegation.md) | Model bounded authority. |
| 6 | [Decision records](../governance/decision-records.md) | Preserve attributable governance decisions. |
| 7 | [Oversight and transparency](../operating-model/oversight-transparency.md) | Make governance reviewable. |
| 8 | [Challenge and redress](../operating-model/challenge-redress.md) | Provide routes to remedy. |
| 9 | [Policy change management](../operating-model/policy-change-management.md) | Govern change over time. |
| 10 | [Worked scenario](../reference-scenario/index.md) | Observe governance operating through a real trace. |

## Reading method

Read the pages in order on the first pass. On later visits, use the sidebar and search as a reference system. The sequence is curated; it does not imply that later pages are normatively subordinate to earlier ones.
