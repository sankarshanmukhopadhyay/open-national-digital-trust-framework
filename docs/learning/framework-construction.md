# Framework construction path

**Audience:** national programme teams, scheme designers, regulators, governance architects, and facilitators  
**Prerequisites:** [ONDTF in One Hour](one-hour.md)  
**Estimated time:** 3–5 hours before workshops  
**Outcome:** understand how to construct, validate, and govern a DTF profile package using ONDTF.

## Reading sequence

| Order | Read | Purpose |
|---:|---|---|
| 1 | [Guided Framework Construction](../adoption/guided-framework-construction.md) | understand the construction method |
| 2 | [Construction Stages](../adoption/construction-stages.md) | understand the decision sequence |
| 3 | [Institutional Operating Model](../governance/institutional-operating-model.md) | identify accountable roles |
| 4 | [Provider Lifecycle](../operations/provider-lifecycle.md) | define participation and status |
| 5 | [Assurance Composition](../assurance/assurance-composition.md) | set defensible assurance thresholds |
| 6 | [Affected-Party Operating Model](../rights/affected-party-operating-model.md) | operationalise rights and remedy |
| 7 | [Profile Methodology](../../profiles/profile-methodology.md) | construct the governed profile |
| 8 | [Contradiction and Completeness](../adoption/contradiction-and-completeness.md) | validate the result |
| 9 | [Workshop Guide](../adoption/workshop-guide.md) | prepare facilitated decision-making |

Use the generated package as a draft governance instrument. It becomes authoritative only through the approval route established by the adopting institution.


## Apply the learning path

After completing the conceptual sequence, inspect the [worked operational profile](../../examples/worked-profile/). Compare each guided answer with the generated operating model, machine-readable registers and validation result. The example intentionally retains one deployment-blocking authority qualification so that learners can see the difference between structural completeness and legitimate adoption.
