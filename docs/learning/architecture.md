# Architecture Path

**Audience:** Enterprise, solution, security, and public-infrastructure architects  
**Prerequisites:** [ONDTF in One Hour](one-hour.md), or equivalent familiarity  
**Estimated time:** 3–4 hours  
**Outcome:** a role-specific understanding of the framework, with enough context to use the reference documentation confidently.

## Recommended sequence

| Order | Read | Why it comes here |
|---:|---|---|
| 1 | [Conceptual baseline](../core-specification/conceptual-baseline.md) | Learn the core trust concepts. |
| 2 | [Architecture principles](../architecture/principles.md) | Understand design constraints. |
| 3 | [Layered reference architecture](../architecture/layered-reference-architecture.md) | Understand system planes and layers. |
| 4 | [Architecture viewpoints](../architecture/viewpoints.md) | See the framework from stakeholder and system perspectives. |
| 5 | [Component catalogue](../architecture/component-catalogue.md) | Understand deployable responsibilities. |
| 6 | [Service catalogue](../architecture/service-catalogue.md) | Understand service capabilities. |
| 7 | [Interaction catalogue](../architecture/interaction-catalogue.md) | Understand exchanges and decision points. |
| 8 | [Information architecture](../information-model/index.md) | Connect components to shared semantics. |
| 9 | [Trust boundaries](../architecture/trust-boundaries.md) | Understand boundary crossings. |
| 10 | [Security architecture](../security/index.md) | Connect architecture to threats and controls. |
| 11 | [Worked scenario](../reference-scenario/end-to-end-walkthrough.md) | Validate the design end to end. |

## Reading method

Read the pages in order on the first pass. On later visits, use the sidebar and search as a reference system. The sequence is curated; it does not imply that later pages are normatively subordinate to earlier ones.
