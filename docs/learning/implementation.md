# Implementation Path

**Audience:** Programme delivery teams, scheme operators, platform teams, and integrators  
**Prerequisites:** [ONDTF in One Hour](one-hour.md), or equivalent familiarity  
**Estimated time:** 3–4 hours  
**Outcome:** a role-specific understanding of the framework, with enough context to use the reference documentation confidently.

## Recommended sequence

| Order | Read | Why it comes here |
|---:|---|---|
| 1 | [Scope and non-goals](../core-specification/scope-and-non-goals.md) | Bound the implementation. |
| 2 | [Capability model](../core-specification/capability-model.md) | Define required outcomes. |
| 3 | [Target operating model](../operating-model/index.md) | Establish institutional responsibilities. |
| 4 | [Implementation guide](../implementation/index.md) | Follow the recommended sequence. |
| 5 | [Reference architecture](../implementation/reference-architecture.md) | Translate the framework into a deployment. |
| 6 | [Canonical workflows](../workflows/index.md) | Implement lifecycle and trust-resolution flows. |
| 7 | [Conformance](../implementation/conformance.md) | Define implementation claims. |
| 8 | [Evidence package](../reference-scenario/evidence-package.md) | See reusable example artefacts. |
| 9 | [Adoption](../adoption/index.md) | Plan staged rollout and scaling. |

## Reading method

Read the pages in order on the first pass. On later visits, use the sidebar and search as a reference system. The sequence is curated; it does not imply that later pages are normatively subordinate to earlier ones.
