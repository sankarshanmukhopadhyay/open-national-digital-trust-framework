# Conformance Framework

Conformance is claimed against a named profile and version.

## Conformance classes

- issuer;
- wallet or holder agent;
- verifier;
- registry or trust resolver;
- policy engine;
- evidence service;
- governance authority;
- assurance provider.

A conformance suite SHOULD include positive, negative, malformed, replay, stale-status, privacy, accessibility, recovery and cross-vendor interoperability tests.

No implementation may claim “ONDTF compliant” without naming the tested class, profile, version and assurance scope.
