# Implementation lessons

The worked scenario demonstrates five practical conclusions. First, authenticated submission is not equivalent to valid delegation. Second, evidence validity depends on issuer competence, provenance, status, semantics, and decision context. Third, an effect must be admitted only after authority and policy checks complete. Fourth, a decision receipt must preserve enough information for an affected party to understand and challenge the outcome. Fifth, correction requires more than changing the final status: the remedy must update the decision, effect, audit record, and assurance view.

The scenario also exposes implementation questions that future profiles must answer, including evidence-retention limits, privacy-preserving status resolution, challenge service levels, cross-domain recognition, and the treatment of partially automated decisions.
