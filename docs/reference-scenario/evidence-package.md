# Evidence package

The scenario publishes reusable fixtures:

- [`authority-record.yaml`](../../examples/reference-scenario/authority-record.yaml)
- [`participant-record.yaml`](../../examples/reference-scenario/participant-record.yaml)
- [`delegation-record.yaml`](../../examples/reference-scenario/delegation-record.yaml)
- [`credential.json`](../../examples/reference-scenario/credential.json)
- [`registry-response.json`](../../examples/reference-scenario/registry-response.json)
- [`policy.json`](../../examples/reference-scenario/policy.json)
- [`decision-receipt.json`](../../examples/reference-scenario/decision-receipt.json)
- [`audit-entry.json`](../../examples/reference-scenario/audit-entry.json)
- [`challenge-request.json`](../../examples/reference-scenario/challenge-request.json)
- [`remedy-decision.json`](../../examples/reference-scenario/remedy-decision.json)

These fixtures are examples, not normative schemas. They are designed to make the framework testable and to expose missing relationships between authority, evidence, policy, decision, effect, accountability, and remedy.
