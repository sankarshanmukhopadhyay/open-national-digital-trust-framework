# Conformance claim types

ONDTF defines three principal claim layers.

| Claim layer | Assessed object | Typical evidence |
|---|---|---|
| Artefact conformance | document, schema, profile or data package | structural and semantic validation |
| Implementation conformance | software, component or deployment | executable tests and security evidence |
| Operational conformance | organisation, provider or service | governance, people, process and operating evidence |

A claim MUST state the claimant, assessed object, applicable requirements, exclusions, profile, assessment method, assessor, decision authority, issue and expiry times, status source and evidence reference.

Self-declaration, second-party assessment, independent assessment, accredited certification and authority approval MUST NOT be represented as equivalent assurance mechanisms.
