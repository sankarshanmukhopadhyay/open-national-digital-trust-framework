# Conformance model

ONDTF conformance is scoped, profile-based, and evidence-backed. A bare statement that a system is “ONDTF compliant” is insufficient.

## Required claim structure

A conformance claim must state:

1. ONDTF version;
2. selected jurisdiction, sector, and technical profiles;
3. conformance class;
4. organisational and service boundary;
5. capabilities implemented;
6. exclusions and inherited controls;
7. assessment method and evidence date;
8. unresolved limitations.

## Initial conformance classes

| Class | Purpose |
|---|---|
| Framework | Governance authority responsible for an ONDTF adoption |
| Trust scheme | Governed multi-party arrangement within the framework |
| Service | Registry, resolver, verifier, issuer, wallet, policy, assurance, or redress service |
| Participant | Organisation or actor admitted under a named profile |
| Profile | Jurisdiction, sector, or technical specification claiming alignment with the core |
| Assessor | Body evaluating controls or conformance evidence |

## Independence rule

Equivalent mechanisms may satisfy the same requirement. A profile may prescribe particular technologies, but the core conformance model evaluates capabilities and outcomes rather than membership in an external repository ecosystem.
