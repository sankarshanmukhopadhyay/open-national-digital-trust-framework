# Future work

Work after v0.5.0 should prioritise maturation rather than adding new foundational domains.

## v0.6.x to v0.8.x

- Independent implementation feedback and issue resolution
- Additional worked deployments and sector profiles
- Interoperability test vectors and conformance fixtures
- Stronger quantitative calibration from operational evidence
- Expanded jurisdiction profiles with qualified legal review
- Accessibility, internationalisation and terminology refinement
- Adversarial exercises and incident simulations
- Performance, scale and recovery testing

## v0.9.0 candidate specification

The candidate specification gate should require stable normative language, multiple independent implementations, resolved high-severity issues, complete traceability and a credible change-control process for the path to v1.0.0.
