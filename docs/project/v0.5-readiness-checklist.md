# v0.5.0 readiness checklist

## Purpose

This checklist is the repository-level integration gate for the ONDTF v0.5.0 Feature Complete Draft. It records what must be true before the version is changed or a release is tagged. Completion means that the intended feature surface is present and internally integrated. It does not imply production certification, legal approval, or evidence from independent deployments.

## Feature-domain completion

- [x] Core specification and capability model are published.
- [x] Governance, authority, delegation, challenge and remedy are represented.
- [x] Reference architecture, information architecture and canonical workflows are published.
- [x] Security objectives, protected assets, boundaries, threats and controls are traceable.
- [x] Operational risk, assurance, continuous monitoring and resilience are documented.
- [x] Data security, privacy, affected-party rights and jurisdiction profiling are documented.
- [x] A worked reference scenario and machine-readable fixtures are available.
- [x] Guided learning paths and role-oriented navigation are available.

## Integration gates

- [x] Every published Markdown page has valid front matter and a stable navigation position.
- [x] Repository-local links resolve from source.
- [x] GitHub Pages publication coverage and generated routes are validated.
- [x] Mermaid source is extracted and parsed in CI before deployment.
- [x] Machine-readable YAML and JSON artefacts parse successfully.
- [x] Standards citations resolve to the consolidated standards register.
- [x] External-reference classifications distinguish foundational, profile-dependent, informative and jurisdiction-specific sources.
- [x] Normative language is governed by the ONDTF normative-language policy.
- [x] Known limitations and unresolved matters are disclosed.

## Release-only actions still pending

The following actions belong in the separate v0.5.0 release commit:

- [ ] Change the framework version from `0.4.0` to `0.5.0` in authoritative version-bearing files.
- [x] Mark the maturity as **Feature Complete Draft**.
- [x] Finalise the changelog and release notes.
- [ ] Confirm a clean GitHub Pages deployment from the release commit.
- [ ] Create and verify the `v0.5.0` tag and GitHub release after merging this release commit.

## Exit decision

The release decision MUST be based on the automated candidate check, review of the [integration review](integration-review.md), and explicit acceptance of the [unresolved issues register](unresolved-issues.md). Any exception MUST be recorded with an owner, rationale, expiry or review trigger, and accepted residual risk.
