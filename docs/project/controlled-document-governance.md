# Controlled document governance

ONDTF specifications, profiles, schemas, registers, mappings, templates, and guidance are controlled artefacts. Each must have an owner, normative status, version, approval route, review cycle, dependencies, and retirement state.

The canonical register is `model/project/controlled-document-register.yaml`. A change to a controlled artefact must be assessed for downstream impact across requirements, profiles, generated packages, conformance evidence, and implementation guidance.
