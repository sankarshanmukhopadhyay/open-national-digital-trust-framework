# Institutional functions

ONDTF specifies functions rather than a mandatory agency structure. One institution may perform several functions where permitted, but the resulting concentration and conflict risks must be assessed.

| Function | Minimum responsibility |
|---|---|
| Framework sponsorship | Establish public purpose, mandate, funding, and political accountability |
| Framework governance | Approve core policy, profiles, decision rights, and material changes |
| Sector governance | Interpret and strengthen the framework for a regulated or specialised domain |
| Trust-scheme governance | Define participant rules, services, assurance, and remedies for a scheme |
| Operations | Run shared services, registries, resolution, monitoring, and support |
| Accreditation or recognition | Determine whether participants or assessors meet admission criteria |
| Assurance and assessment | Evaluate controls, evidence, and continuing conformity |
| Security coordination | Coordinate threat intelligence, incidents, vulnerability response, and recovery |
| Privacy and rights oversight | Examine proportionality, exclusion, surveillance, and affected-party impacts |
| Appeals and redress | Correct records, review decisions, order remedies, and escalate disputes |
| Standards and interoperability | Maintain profiles, mappings, testing, and mutual-recognition arrangements |
| Transparency and evaluation | Publish performance, incidents, decisions, limitations, and review findings |

## Separation principle

An operating model SHOULD avoid allowing the same body to set requirements, operate the service, assess its own compliance, and decide final appeals without independent safeguards.
