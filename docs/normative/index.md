# Normative architecture

ONDTF v0.6.0 work begins by converting the feature-complete conceptual model into a controlled requirements system. This section defines how obligations are written, scoped, assigned, inherited, evidenced, excepted and traced.

The normative architecture is deliberately separate from any single jurisdiction, protocol or product. It provides a stable core that profiles can specialise without silently weakening mandatory framework obligations.

The catalogue is the authoritative inventory for requirement identifiers, applicability, accountable roles and evidence expectations. The Markdown pages explain how to apply it.
